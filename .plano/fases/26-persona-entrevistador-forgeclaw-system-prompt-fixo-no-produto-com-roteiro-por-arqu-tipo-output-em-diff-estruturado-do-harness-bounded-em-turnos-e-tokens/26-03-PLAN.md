# Fase 26 Plano 03: Roteiros por Arquetipo (scripts/*.md)

**Objetivo:** Criar os 5 roteiros (`packages/core/src/onboarding/scripts/<slug>.md`) que guiam o Entrevistador em cada arquetipo. Os roteiros sao anexados ao system prompt fixo via `loadInterviewerPrompt(slug)`. Cada script e uma LISTA PRIORIZADA de topicos que o entrevistador deve tentar cobrir, com pergunta sugerida e chave de diff correspondente — mas o entrevistador mantem a liberdade de pular se o usuario ja respondeu ou de terminar cedo com `done`.

## Contexto

@packages/core/src/onboarding/interviewer.md — system prompt base (regras inegociaveis, formato JSON, placeholders permitidos)
@packages/core/src/onboarding/prompts.ts — loadScript(slug) consome este arquivos
@packages/core/src/onboarding/types.ts — VALID_PLACEHOLDER_KEYS, HarnessFile, DiffOp
@packages/cli/src/templates/archetypes/solo-builder/archetype.json — suggestedAgents e recommendedTools do arquetipo
@packages/cli/src/templates/archetypes/content-creator/archetype.json — idem
@packages/cli/src/templates/archetypes/agency-freela/archetype.json — idem
@packages/cli/src/templates/archetypes/ecom-manager/archetype.json — idem
@packages/cli/src/templates/archetypes/generic/archetype.json — idem

## Regras transversais

1. **Portugues brasileiro.** Termos tecnicos em ingles quando natural.
2. **Topicos priorizados.** Cada script lista entre 4 e 8 topicos. Os primeiros 4 SEMPRE cobrem: userName, company, role, workingDir (ou vaultPath, se o arquetipo depende mais de vault). Os seguintes sao especificos do perfil.
3. **Cada topico tem:** (a) objetivo em 1 linha, (b) pergunta sugerida, (c) acao de diff esperada (qual arquivo, qual op).
4. **Nao invente secoes inexistentes nos templates.** Se quiser alterar uma secao H2, verifique que ela existe no template de Fase 24 (ex: `## Pilares editoriais` existe em `content-creator/USER.md`).
5. **Script e guia, nao contrato.** Comece o script com um preambulo curto deixando claro que o entrevistador pode pular topicos, encerrar cedo ou reordenar.

# Roteiros do Entrevistador

Um arquivo por `ArchetypeSlug`. Cada script e anexado ao system prompt fixo
(`../interviewer.md`) em runtime via `loadInterviewerPrompt(slug)`.

## Arquivos esperados

- `solo-builder.md`
- `content-creator.md`
- `agency-freela.md`
- `ecom-manager.md`
- `generic.md`

Se um arquivo nao existir pro slug pedido, `loadInterviewerPrompt` retorna o
prompt base sozinho (com warning) — nao joga erro.

## Estrutura obrigatoria de cada script

1. **Titulo:** `# Roteiro: <Nome legivel>` (linha 1).
2. **Regras especificas deste arquetipo:** 2-4 bullets com contexto especifico (tom, caveats, sensibilidades).
3. **Topicos priorizados:** lista numerada (`### 1.`, `### 2.`, ...) com entre 4 e 8 topicos.
4. **Sinal de `done` antecipado:** paragrafo final explicando em quais topicos o entrevistador ja pode terminar.

## Schema de cada topico

```
### N. Titulo curto
- **Objetivo:** o que capturar (placeholder ou secao H2)
- **Pergunta sugerida:** uma frase, max ~150 chars
- **Diff:** `<op> <key ou header> <file>` — instrucao compacta de que diff emitir
- **Nota:** (opcional) caveat sobre createIfMissing / placeholder nao universal / etc.
```

## Regras de content
