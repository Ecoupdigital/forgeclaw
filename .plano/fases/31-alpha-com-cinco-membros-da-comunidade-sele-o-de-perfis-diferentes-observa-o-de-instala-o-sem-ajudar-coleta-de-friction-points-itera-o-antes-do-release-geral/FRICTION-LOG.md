# Friction Log — Alpha Fase 31

Lista de friction points — coisas que NAO sao bug mas que irritam o usuario, confundem ou atrapalham o fluxo. Diferente de bug, **friction NAO tem SLA de fix durante o alpha**. Acumula aqui e Jonathan avalia no PLAN 31-03 quais viram P0/P1/P2 para o release geral.

## O que conta como friction (nao-bug)

- Mensagem de erro confusa (o produto funciona mas o humano nao entende)
- Fluxo contra-intuitivo (precisa clicar em 3 coisas diferentes pra fazer algo simples)
- Nome/label ruim (botao "Processar" quando deveria ser "Enviar")
- Falta de feedback visual (clica e nada parece ter acontecido)
- Onboarding vague (pergunta que nao explica o que fazer com a resposta)
- Documentacao ambigua (README promete X mas X esta em outro lugar)
- Performance percebida (funciona mas parece lento)

## O que NAO e friction (e bug — vai pro BUG-TRIAGE)

- Produto crasha
- Feature nao responde
- Dado nao salva
- Erro nao tratado aparece no console
